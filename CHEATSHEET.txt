🆘 快速故障排查
───────────────────────────────────────────────────────────────
问题：GROQ_API_KEY 未设置
解决：nano .env （填入真实密钥）

问题：ffmpeg 未安装
解决：brew install ffmpeg

问题：下载失败（yt-dlp 错误）
解决：pip install --upgrade yt-dlp

问题：OCR 处理很慢
解决：使用 mobile 模型（默认）或减少抽帧频率

问题：OCR 识别不准确
解决：使用 server 模型提高精度
      make ocr VIDEO=xxx DET_MODEL=server REC_MODEL=server

问题：虚拟环境损坏
解决：make clean-all && make test （清理并重建）

问题：内存不足
解决：使用 mobile 模型，或减少同时处理的帧数

问题：首次运行很慢
说明：正在自动下载模型和依赖，后续会很快

问题：B站下载失败
解决：安装 BBDown: brew install bbdown

💡 Pro Tips
───────────────────────────────────────────────────────────────
1. 🆕 首次使用直接开始，无需手动配置：
   make run VIDEO=video.mp4  # 自动完成所有环境配置

2. 🆕 从URL一键下载并处理：
   make download-run URL="https://www.youtube.com/watch?v=xxx"
   make download-ocr URL="https://www.bilibili.com/video/BVxxx"

3. 🆕 支持多个平台的视频，统一处理流程：
   make download-ocr URL="https://example.com/video"
   # 支持 YouTube, Bilibili, 小红书, 抖音, Twitter 等

4. OCR模式带进度条，实时查看处理进度：
   make ocr VIDEO=xxx.mp4  # 显示"🔍 OCR进度: 52/52"

5. 批量处理多个视频：
   for f in *.mp4; do make run VIDEO="$f"; done

6. 使用绝对路径避免问题：
   make run VIDEO="$(pwd)/video.mp4"

7. 查看实时处理日志：
   make run VIDEO=xxx.mp4 2>&1 | tee process.log

8. 自定义输出目录：
   python process_video.py video.mp4 --out-dir /custom/path

9. 根据设备性能选择 OCR 模型：
   # 普通笔记本 → mobile（默认）
   # 高性能工作站 → server
   # 平衡模式 → mobile检测 + server识别

10. OCR流程优化 - 先OCR后音频：
    # 如果OCR识别有问题，可以及早发现
    # 不会浪费时间在音频转写上

11. 清理并重置环境：
    make clean-all  # 删除虚拟环境和输出文件
    make test       # 重新创建环境并测试

12. 查看环境状态：
    make check      # 显示虚拟环境、配置、依赖等状态

💡 选择建议：
• 简单页面、不需要登录 → Crawl4AI (自动)
• 需要登录、复杂JS渲染 → DrissionPage (自动)
• 反爬严格（小红书、推特）→ DrissionPage (自动)
• 需要纯净内容（无评论）→ MODE=default
• 需要完整内容（含评论）→ MODE=full

🔧 Cookie配置详解
───────────────────────────────────────────────────────────────
小红书Cookie获取方法：
1. 浏览器访问：https://www.xiaohongshu.com/explore
2. 登录账号
3. F12打开开发者工具
4. Network标签 → 刷新页面
5. 找到任意请求 → Request Headers
6. 复制完整的Cookie值
7. 运行 make config-xhs-cookie 粘贴

Cookie优先级：
1. XHS-Downloader配置（推荐，自动）
2. 浏览器Cookie（--browser参数）
3. Cookie文件（--cookies参数）

Cookie验证：
  python scripts/test_cookie_unified.py
  # 检查Cookie是否配置成功

🛠️ 故障排查
───────────────────────────────────────────────────────────────
问题：提示 Crawl4AI 未安装
解决：
  pip install crawl4ai
  playwright install chromium

问题：小红书归档失败
解决：
  1. 配置Cookie：make config-xhs-cookie
  2. 验证配置：python scripts/test_cookie_unified.py
  3. 或手动指定：python -m cli.archive_cli URL --browser chrome

问题：内容提取不完整
解决：
  # 使用调试模式查看
  python -m cli.archive_cli URL --show-browser -v

问题：403错误
解决：
  1. 配置Cookie（如小红书）
  2. 检查User-Agent
  3. 降低并发数：-c 1

问题：批量归档部分失败
说明：
  # 正常现象，会显示失败的URL
  # 可以单独重试失败的URL

📊 性能建议
───────────────────────────────────────────────────────────────
• 单个归档：约3-10秒（取决于网页复杂度）
• 批量归档：建议并发数3-5
• Cookie有效期：定期更新（通常几周到几个月）
• 归档频率：避免过于频繁请求同一网站

🎯 高级用法（Python API）
───────────────────────────────────────────────────────────────
from db import SearchRepository

repo = SearchRepository()

# 全文搜索
results = repo.search(
    query="机器学习",
    tags=["教育", "科技"],
    fields='transcript',      # report/transcript/ocr/all
    sort_by='relevance',      # relevance/date/duration
    limit=20
)

# 处理结果
for result in results:
    print(f"视频: {result.video_title}")
    print(f"时间: {result.timestamp_seconds}s")
    print(f"片段: {result.matched_snippet}")
    print(f"文件: {result.file_path}")

🔧 故障排查 - 数据库
───────────────────────────────────────────────────────────────
问题：数据库文件不存在
解决：make db-init

问题：搜索无结果
解决：
  1. 确认已处理视频并入库
  2. 检查搜索关键词是否正确
  3. 尝试简化搜索词

问题：数据库锁定
解决：
  sqlite3 storage/database/knowledge.db "PRAGMA wal_checkpoint;"

问题：搜索很慢
解决：
  1. make db-vacuum  # 优化数据库
  2. 减少返回结果数量（--limit 参数）

问题：想清空数据库重新开始
解决：make db-reset

🚀 启动与配置
───────────────────────────────────────────────────────────────
  make tg-bot-setup          # 首次配置：设置 Token 和 User ID
  make tg-bot-start          # 启动 Bot（后台运行）
  make tg-bot-stop           # 停止 Bot
  make tg-bot-log            # 查看运行日志

💬 Bot 命令列表
───────────────────────────────────────────────────────────────
┌─ 自动归档（最常用）─────────────────────────────────────────┐
│ 直接发送 http 链接         → 自动识别并归档到知识库          │
│ 支持：网页、B站、YouTube、小红书、抖音、Twitter 等          │
└─────────────────────────────────────────────────────────────┘

┌─ 搜索与查看────────────────────────────────────────────────┐
│ /search <关键词>           全文搜索知识库                   │
│ /db_list                  列出前 20 条记录                  │
│ /db_list <页码>            翻页（例：/db_list 2）           │
│ /db_show <ID>             查看记录详情（例：/db_show 43）   │
│ /db_stats                 知识库存储统计                    │
└─────────────────────────────────────────────────────────────┘

┌─ 高级处理──────────────────────────────────────────────────┐
│ /download_run <URL>        下载视频 + 音频分析               │

│ /archive_run <URL>         完整归档长网页 + AI 报告          │
│ /archive_ocr <URL>         归档网页 + 版面 OCR               │
└─────────────────────────────────────────────────────────────┘

┌─ URL 工具──────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────┘

🔗 URL 反追踪 & 短链接还原 - 快速参考
═══════════════════════════════════════════════════════════════

🚀 基本用法
───────────────────────────────────────────────────────────────
  make url-clean URL="链接"                  # 清理单个 URL
  make url-clean URL="含链接的文字 https://…" # 自动从文本提取 URL

  # Bot 中使用：
  /url_clean <URL或包含URL的文本>

🔄 短链接展开（自动识别以下域名）
───────────────────────────────────────────────────────────────
  t.co        bit.ly      tinyurl.com   goo.gl
  xhslink.com b23.tv      tb.cn         url.cn
  是否展开判断逻辑：
    1. 域名在白名单（如 zhihu.com）→ 跳过展开，直接清理参数
    2. 已知短链接域名 → 强制展开
    3. 路径 ≤ 15 字符的未知域名 → 视为短链接展开

🧹 去除追踪参数（各平台）
───────────────────────────────────────────────────────────────
  通用：     utm_source  utm_medium  utm_campaign  utm_content
             gclid  fbclid  msclkid  ref  referrer

  淘宝/天猫：spm  ut_sk  share_source  shareUniqueId
  京东：     重定向自动跳转 → 还原商品页，保留 product ID
  抖音：     自动提取 /video/<ID> 保留纯净链接
  网易云：   自动提取 song?id= 保留纯净链接
  小红书：   删除 xsec_token  xsec_source  xhsshare  apptime 等
  B站：      bbid  ts  share_tag  share_session_id
  知乎：     utm_oi  utm_psn  share_code
  微信/通用：scene  subscene  sessionid  clicktime  enterid
  Instagram：igshid  story_media_id

📝 示例
───────────────────────────────────────────────────────────────
  输入：https://zhihu.com/question/123?utm_source=wechat&share_code=abc
  输出：https://zhihu.com/question/123

  输入：https://www.xiaohongshu.com/explore/abc?xsec_token=xxx&xsec_source=yyy
  输出：https://www.xiaohongshu.com/explore/abc

  输入：https://b23.tv/abc123   （短链接）
  输出：展开后的 B 站完整链接（自动去除追踪参数）
